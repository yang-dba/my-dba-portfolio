# 故障升级矩阵

## MySQL 故障升级矩阵

### 一、故障等级定义

| 等级   | 名称     | 典型场景                                                     | 响应时限 | 升级时限 |
| ------ | -------- | ------------------------------------------------------------ | -------- | -------- |
| **P0** | 紧急故障 | 数据库宕机、数据丢失、主从全部故障、无法连接                 | 即时     | 5分钟    |
| **P1** | 严重故障 | 主从延迟>1小时、大量慢查询占满CPU、连接数爆满、业务大面积超时 | 10分钟   | 30分钟   |
| **P2** | 一般故障 | 单从库延迟、磁盘空间<10%、死锁增多、备份失败                 | 30分钟   | 2小时    |
| **P3** | 轻微故障 | 慢查询偶发、告警阈值波动、非核心报表延迟                     | 当天处理 | 无需升级 |

---

### 二、升级路径

#### P0 级别（紧急故障）

| 时间点    | 角色       | 行动                                               |
| --------- | ---------- | -------------------------------------------------- |
| 0-5分钟   | 值班DBA    | 立即执行应急预案（重启/切流/恢复），同步研发负责人 |
| 5-15分钟  | DBA Leader | 介入协调，评估是否需要回滚或切备库，通报技术总监   |
| 15-30分钟 | 技术总监   | 通报管理层，协调跨团队资源，对外发布故障公告       |
| >30分钟   | CTO/VP     | 启动公司级事故处理流程，全员通报，成立应急小组     |

#### P1 级别（严重故障）

| 时间点    | 角色       | 行动                                                       |
| --------- | ---------- | ---------------------------------------------------------- |
| 0-10分钟  | 值班DBA    | 定位问题，尝试常规手段恢复（kill会话、调整参数、重启从库） |
| 10-30分钟 | DBA Leader | 评估影响，指导修复方案，同步研发负责人                     |
| 30-60分钟 | 研发负责人 | 确认业务影响，准备客户公告，协调应用降级                   |
| >60分钟   | 技术总监   | 升级为P0处理，启动应急响应流程                             |

#### P2/P3 级别

| 时间点       | 角色       | 行动                                   |
| ------------ | ---------- | -------------------------------------- |
| 30分钟-2小时 | 值班DBA    | 记录问题，排期处理，更新工单状态       |
| 当日内       | DBA Leader | 确认修复方案，纳入迭代计划，更新知识库 |

### 七、注意事项

1. **升级不是追责**：超过时限主动升级是流程要求，不是个人能力问题，目的是尽快恢复业务
2. **信息同步**：每次升级必须同步当前处理进展、已尝试的方案、需要的支持
3. **文档记录**：所有故障处理过程需在 `incident-response/` 下归档，形成复盘报告
4. **定期演练**：每季度组织一次 P0 故障模拟演练，验证流程有效性
5. **联系方式维护**：联系人矩阵每季度更新一次，确保准确

### 八、升级流程示意图

```text
故障发生
   │
   ▼
值班DBA响应（0-5分钟）
   │
   ├── 自己能解决 ──▶ 关闭工单，记录复盘
   │
   └── 不能解决或超时
         │
         ▼
    升级至 DBA Leader（5-15分钟）
         │
         ├── 解决 ──▶ 记录复盘，优化应急预案
         │
         └── 仍未解决
               │
               ▼
          升级至 技术总监（15-30分钟）
               │
               ├── 解决 ──▶ 通报管理层，复盘
               │
               └── 重大故障
                     │
                     ▼
                升级至 CTO/VP（>30分钟）
                     │
                     ▼
                启动公司级应急响应
```
