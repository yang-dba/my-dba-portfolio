# MySQL 监控阈值参考表

## 一、性能指标

| 指标                 | 正常范围       | 警告阈值    | 紧急阈值    | 说明                       | 处理建议                                         |
| -------------------- | -------------- | ----------- | ----------- | -------------------------- | ------------------------------------------------ |
| QPS（每秒查询数）    | 与业务基线对比 | 突增 > 200% | 突增 > 500% | 每秒查询数，观察波动       | 突增可能由流量上升或慢查询导致，需排查           |
| TPS（每秒事务数）    | 与业务基线对比 | 下降 > 50%  | 下降 > 80%  | 每秒事务数，主要针对写入   | 下降可能表示锁竞争或主从延迟                     |
| InnoDB 缓冲池命中率  | > 99%          | < 98%       | < 95%       | 数据页在内存中的命中率     | 命中率低需增加 `innodb_buffer_pool_size`         |
| InnoDB 数据读/写请求 | 与基线对比     | 突增 > 100% | 突增 > 200% | `Innodb_data_reads/writes` | 突增可能表示索引失效或全表扫描                   |
| 临时表磁盘使用率     | 0%             | > 20%       | > 50%       | 创建磁盘临时表的比例       | 出现磁盘临时表应优化 SQL 或增加 `tmp_table_size` |
| 表锁等待             | 0              | > 10/s      | > 50/s      | 表级锁等待次数             | 出现表锁需检查是否使用 MyISAM 引擎               |

## 二、连接与线程

| 指标         | 正常范围                | 警告阈值                | 紧急阈值                | 说明                 | 处理建议                                        |
| ------------ | ----------------------- | ----------------------- | ----------------------- | -------------------- | ----------------------------------------------- |
| 当前连接数   | < max_connections * 0.7 | > max_connections * 0.8 | > max_connections * 0.9 | 当前活跃连接数       | 接近上限需增加 `max_connections` 或排查空闲连接 |
| 活跃线程数   | < 50                    | > 100                   | > 200                   | 正在执行查询的线程数 | 过高可能由慢查询阻塞导致                        |
| 创建线程数   | 0                       | > 10/min                | > 50/min                | 每秒创建的新线程数   | 频繁创建表示线程池不足或连接抖动                |
| 连接失败率   | < 1%                    | > 5%                    | > 20%                   | 连接失败比例         | 检查网络、认证、`max_connections` 限制          |
| 慢查询并发数 | < 5                     | > 10                    | > 30                    | 同时执行的慢查询个数 | 并发慢查询会耗尽资源，需立即优化                |

## 三、复制与高可用

| 指标             | 正常范围 | 警告阈值 | 紧急阈值 | 说明                    | 处理建议                                |
| ---------------- | -------- | -------- | -------- | ----------------------- | --------------------------------------- |
| 主从延迟（秒）   | < 10     | > 60     | > 300    | `Seconds_Behind_Source` | 延迟增大需检查大事务或从库硬件          |
| IO 线程状态      | Yes      | No       | No       | `Replica_IO_Running`    | 若为 No，检查网络和复制用户权限         |
| SQL 线程状态     | Yes      | No       | No       | `Replica_SQL_Running`   | 若为 No，检查 SQL 错误并跳过或修复      |
| 中继日志空间     | < 1GB    | > 10GB   | > 50GB   | `Relay_Log_Space`       | 过大表示从库回放严重落后                |
| 主从 GTID 差异   | 0        | > 10     | > 100    | 主库与从库 GTID 集合差  | 差异过大需重建从库                      |
| 主库 binlog 空间 | < 10GB   | > 50GB   | > 100GB  | binlog 文件总大小       | 过大需调整 `binlog_expire_logs_seconds` |

## 四、存储与磁盘

| 指标                | 正常范围   | 警告阈值      | 紧急阈值      | 说明                         | 处理建议                      |
| ------------------- | ---------- | ------------- | ------------- | ---------------------------- | ----------------------------- |
| 数据目录磁盘使用率  | < 70%      | > 80%         | > 90%         | `/data` 分区使用率           | 清理 binlog、归档数据、扩容   |
| 日志目录磁盘使用率  | < 70%      | > 80%         | > 90%         | `/var/log` 分区使用率        | 清理慢查询日志和错误日志      |
| InnoDB 数据文件大小 | 与基线对比 | 增长 > 20%/月 | 增长 > 50%/月 | `ibdata1` 及各表 `.ibd` 大小 | 检查是否有异常大表            |
| Undo 表空间大小     | < 1GB      | > 2GB         | > 5GB         | `undo_001` `undo_002` 大小   | 过大表示存在长事务            |
| 临时表空间大小      | < 1GB      | > 5GB         | > 10GB        | `ibtmp1` 大小                | 过大需排查磁盘临时表产生原因  |
| 碎片率              | < 10%      | > 20%         | > 50%         | `DATA_FREE / DATA_LENGTH`    | 高碎片需执行 `OPTIMIZE TABLE` |

---

## 五、慢查询与 SQL

| 指标                   | 正常范围  | 警告阈值  | 紧急阈值   | 说明                                          | 处理建议                   |
| ---------------------- | --------- | --------- | ---------- | --------------------------------------------- | -------------------------- |
| 慢查询数量（每分钟）   | < 10      | > 50      | > 200      | 每分钟慢查询计数                              | 需抓取慢日志分析           |
| 慢查询最长耗时         | < 2s      | > 5s      | > 10s      | 单条慢查询最大执行时间                        | 超过阈值需立即优化         |
| 全表扫描次数           | < 100/min | > 500/min | > 1000/min | `Select_scan` 计数器                          | 检查未使用索引的查询       |
| 索引使用率             | > 99%     | < 95%     | < 90%      | `Handler_read_next` / `Handler_read_rnd_next` | 索引命中率低需优化索引设计 |
| 临时表创建次数（磁盘） | < 50/min  | > 200/min | > 500/min  | `Created_tmp_disk_tables`                     | 磁盘临时表严重消耗 I/O     |

## 六、锁与死锁

| 指标           | 正常范围 | 警告阈值 | 紧急阈值 | 说明                            | 处理建议                 |
| -------------- | -------- | -------- | -------- | ------------------------------- | ------------------------ |
| 当前锁等待数   | 0        | > 5      | > 20     | `Innodb_row_lock_current_waits` | 存在行锁等待，需排查     |
| 锁等待超时次数 | < 5/day  | > 10/day | > 50/day | `Innodb_row_lock_timeouts`      | 频繁超时需优化事务       |
| 死锁次数       | 0        | > 5/day  | > 20/day | `Innodb_deadlocks`              | 出现死锁需分析事务顺序   |
| 平均锁等待时间 | < 100ms  | > 500ms  | > 2s     | `Innodb_row_lock_avg`           | 等待时间长表示热点行竞争 |

## 七、错误与告警

| 指标               | 正常范围 | 警告阈值 | 紧急阈值  | 说明                                | 处理建议                     |
| ------------------ | -------- | -------- | --------- | ----------------------------------- | ---------------------------- |
| 错误日志新增错误数 | 0        | > 5/hour | > 20/hour | 每小时错误日志行数                  | 检查错误内容及时处理         |
| 主从错误           | 0        | 任何     | 任何      | `Last_IO_Error` / `Last_SQL_Error`  | 立即排查修复                 |
| 连接超限           | 0        | 任何     | 任何      | `Connection_errors_max_connections` | 需立即增加 `max_connections` |
| 表损坏             | 0        | 任何     | 任何      | `CHECK TABLE` 结果                  | 需立即修复或从备份恢复       |

---

## 八、告警级别说明

| 级别     | 颜色 | 含义                           | 响应要求                  |
| -------- | ---- | ------------------------------ | ------------------------- |
| **正常** | 绿色 | 所有指标在正常范围内           | 无需处理                  |
| **警告** | 黄色 | 指标达到警告阈值，需关注       | 记录日志，排期处理        |
| **紧急** | 红色 | 指标达到紧急阈值，可能影响业务 | 立即介入，启动应急流程    |
| **严重** | 深红 | 已出现业务故障                 | 按 P0/P1 故障升级矩阵处理 |
