# DBA 每日早晨巡检清单

## Mysql

### 一、巡检目标

- 确保数据库服务整体健康
- 提前发现潜在风险（磁盘、慢查询、复制异常等）
- 记录基线数据，便于趋势分析

---

### 二、巡检频率

- **每日一次**：上班后 30 分钟内完成
- 总耗时：约 10-15 分钟

---

### 三、巡检项清单

#### 3.1 系统层面

| 检查项     | 命令/方法                       | 正常标准                | 异常处理                     |
| ---------- | ------------------------------- | ----------------------- | ---------------------------- |
| 服务器负载 | `uptime`、`top -bn1 \| head -5` | load < CPU 核心数 * 0.7 | 检查 CPU 占用高的进程        |
| 内存使用   | `free -h`                       | 可用内存 > 10%          | 排查内存泄漏，考虑扩容       |
| 磁盘空间   | `df -h`                         | 使用率 < 80%            | 清理日志、binlog，或扩容     |
| 磁盘 I/O   | `iostat -x 1 3`                 | `%util` < 80%           | 检查是否有慢查询或大量临时表 |
| 系统日志   | `dmesg -T \| tail -20`          | 无 OOM、磁盘错误        | 根据错误类型处理             |

---

#### 3.2 MySQL 服务状态

| 检查项     | 命令/方法                      | 正常标准           | 异常处理                 |
| ---------- | ------------------------------ | ------------------ | ------------------------ |
| 服务进程   | `systemctl status mysqld`      | active (running)   | 启动失败则恢复并分析原因 |
| 端口监听   | `ss -tlnp \| grep 3306`        | 端口 LISTEN        | 检查 my.cnf 配置、防火墙 |
| 错误日志   | `tail -50 /var/log/mysqld.log` | 无新增 ERROR/FATAL | 根据错误日志排查         |
| 数据库连接 | `mysqladmin ping`              | mysqld is alive    | 连接失败检查权限和网络   |

---

#### 3.3 性能指标

| 检查项       | 命令/方法                                                    | 正常标准                | 异常处理                              |
| ------------ | ------------------------------------------------------------ | ----------------------- | ------------------------------------- |
| 连接数       | `SHOW STATUS LIKE 'Threads_connected';`                      | < max_connections * 0.7 | 排查空闲连接或增加 max_connections    |
| 慢查询数     | `SHOW STATUS LIKE 'Slow_queries';`                           | 较昨日无明显增长        | 分析慢查询日志                        |
| QPS/TPS      | `SHOW STATUS WHERE Variable_name IN ('Questions','Com_commit','Com_rollback');` | 与业务基线对比          | 突增突降需排查业务变更                |
| 缓冲池命中率 | `SHOW ENGINE INNODB STATUS\G` 或计算 `(Innodb_buffer_pool_read_requests - Innodb_buffer_pool_reads) / Innodb_buffer_pool_read_requests` | > 99%                   | 命中率低需增加 buffer pool 或优化查询 |
| 临时表使用   | `SHOW STATUS LIKE 'Created_tmp_disk_tables';`                | 今日增量 < 100          | 增加 `tmp_table_size` 或优化 SQL      |

---

#### 3.4 复制状态（如有主从）

| 检查项           | 命令/方法                                                    | 正常标准               | 异常处理                           |
| ---------------- | ------------------------------------------------------------ | ---------------------- | ---------------------------------- |
| 复制线程         | `SHOW REPLICA STATUS\G` 查看 `Replica_IO_Running` / `Replica_SQL_Running` | Yes                    | 查看 Last_Error，尝试修复或重建    |
| 主从延迟         | `Seconds_Behind_Source`                                      | < 10                   | 延迟增大排查大事务、网络、从库硬件 |
| 中继日志空间     | `Relay_Log_Space`                                            | 稳定或缓慢增长         | 异常增长检查从库回放阻塞           |
| 主从 GTID 一致性 | 主从 `gtid_executed` 对比                                    | 从库 GTID 是主库的子集 | 不一致需重建从库                   |

---

#### 3.5 备份与恢复

| 检查项       | 命令/方法                     | 正常标准                    | 异常处理                       |
| ------------ | ----------------------------- | --------------------------- | ------------------------------ |
| 最近备份     | 查看备份日志 / 备份文件时间戳 | 24 小时内存在成功备份       | 立即执行手动备份，排查备份脚本 |
| 备份文件大小 | `ls -lh` 对比历史             | 无明显异常缩小              | 备份不完整需重新备份           |
| 备份验证     | 定期恢复测试（每周）          | 恢复成功                    | 修复备份脚本                   |
| binlog 保留  | `SHOW BINARY LOGS;`           | 保留周期符合策略（如 7 天） | 清理过期 binlog                |

---

#### 3.6 告警与事件

| 检查项     | 命令/方法                       | 正常标准       | 异常处理                  |
| ---------- | ------------------------------- | -------------- | ------------------------- |
| 监控告警   | 查看钉钉/邮件/企微告警          | 无未处理的告警 | 确认告警原因并处理        |
| 定时任务   | `crontab -l` 检查备份、巡检脚本 | 执行正常       | 手动执行并排查失败原因    |
| 事件调度器 | `SHOW EVENTS;`                  | 启用且执行正常 | 检查 event_scheduler 状态 |

---

#### 3.7 安全与权限

| 检查项       | 命令/方法                                              | 正常标准         | 异常处理               |
| ------------ | ------------------------------------------------------ | ---------------- | ---------------------- |
| 异常登录尝试 | `grep "Access denied" /var/log/mysqld.log \| tail -20` | 无异常           | 排查是否攻击或错误密码 |
| 用户权限变更 | 对比昨日授权                                           | 无未经审批的变更 | 审计并回滚             |
| 弱口令检查   | 定期扫描                                               | 无弱口令         | 通知修改密码           |
